# Phase 2: DPIA

Load `../references/dpia_methodology.md` before running this phase.

## Goal

Determine whether a DPIA is required and, if so, structure the output.

## Ask

- Is there systematic monitoring?
- Is special-category data processed at scale?
- Is there automated decision-making with legal or similar effects?
- Which WP29 high-risk criteria apply?

## Deterministic Path

Prefer:

- `scripts/dpia_generator.py`

Use the template path when building a full report.

## Output

Either:

- a clear DPIA-not-required rationale, or
- a DPIA report with risks, mitigations, and consultation needs
