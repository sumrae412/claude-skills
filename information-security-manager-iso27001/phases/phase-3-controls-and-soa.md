# Phase 3: Controls and Statement of Applicability

Load `../references/iso27001-controls.md` before running this phase.

## Goal

Map risks to controls and define Statement of Applicability decisions.

## Ask

- Which controls are already implemented?
- Which controls are missing or weak?
- Which exclusions need explicit justification?
- What evidence exists for each control domain?

## Deterministic Path

Prefer:

- `scripts/compliance_checker.py`

Use it for baseline compliance or gap-analysis output.

## Build

The deliverable should cover:

- selected controls
- excluded controls with rationale
- control owners
- evidence expectations
- remediation priorities

## Output

Control and SoA guidance with gaps, priorities, and evidence needs.
