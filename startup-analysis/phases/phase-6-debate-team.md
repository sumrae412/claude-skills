# Phase 6: Debate-Team Validation

## Goal

Run the five-role debate team to produce concrete evidence the verdict
in Phase 7 will score against. The five Pass / Hold calls are the
structural input to Go / Conditional Go / No-Go.

## How To Run

Invoke the `startup-debate-team` skill in sequence, passing forward
Phase 1-5 outputs as context:

1. `startup-debate-team/roles/01-researcher.md` — validates pain with
   real-source quotes.
2. `startup-debate-team/roles/02-strategist.md` — 5 ranked lead magnets,
   picks one.
3. `startup-debate-team/roles/03-copywriter.md` — headline / bullets /
   CTA echoing verbatim pain.
4. `startup-debate-team/roles/04-builder.md` — one-page site spec.
5. `startup-debate-team/roles/05-marketer.md` — 30/60/90 funnel with
   visitor → MRR math.

Each role ends with a Pass / Hold call. Capture all five.

## Halting Rules (structural No-Go signals)

Any of the following forces a No-Go in Phase 7, regardless of scorecard:

- **Researcher Hold** — fewer than 2 of 5 pains have verbatim customer
  language. The pain is not real enough to validate.
- **Strategist Hold after two attempts** — cannot design an offer the
  customer would pay $50-$100 for. The wedge has no commercial pull.
- **Marketer Hold on math** — the visitor → MRR math cannot reach the
  target at any defensible ARPU or conversion rate. The unit economics
  do not exist.

## Inputs To Pass Forward

From Phase 1-5, hand the debate team:

- ICP (Phase 2)
- sharpest pain framing (Phase 1 + Phase 2 frustrations)
- wedge / real enemy (Phase 4)
- GTM mode (Phase 5)

## Output

A validation packet containing:

- 5 Pass / Hold calls with one-line reasons
- Researcher's top 5 pains table
- Chosen lead magnet
- Headline + CTA
- One-page site spec
- 30 / 60 / 90 plan + visitor → MRR math
- Any Halting Rule trips, flagged for Phase 7

## Strategist's Note

Close with a one-paragraph synthesis: did the debate team strengthen or
weaken the case from Phases 1-5? Where did it expose a gap the
CLEARFRAME pass missed?
