# Phase 3: Confusion Management

## Goal

Handle conflicts, missing requirements, and ambiguity without silent
guessing.

## Use When

- spec conflicts with code
- requirements are incomplete
- there is precedent ambiguity
- the agent is about to invent behavior

## Patterns

- surface conflicts explicitly
- stop and ask when requirements are truly missing
- emit lightweight inline plans before multi-step execution

## Output

Conflict or ambiguity handling plan for the current task.
