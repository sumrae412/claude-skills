# Phase 4: Verification and Refresh

## Goal

Confirm that the context setup is working and refresh it when it drifts.

## Check

- rules file exists and is relevant
- outputs follow project patterns
- cited APIs and files are real
- context is refreshed when switching major areas
- known gotchas are injected when needed

## Refresh Triggers

- stale or contradictory answers
- wrong conventions recurring
- long session drift
- switching to a different subsystem

## Output

Verification checklist and recommended refresh action.
