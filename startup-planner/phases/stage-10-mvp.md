# Stage 10: MVP Definition and Requirements

Load `../references/business-frameworks.md` before running this stage.

## Goal

Define the minimum viable product, core hypothesis, and first feature
set.

## Ask

- What single thing must be true for this to work?
- What is the smallest version that tests that hypothesis?
- Which MVP pattern fits best?

MVP patterns:

- Concierge MVP
- Wizard of Oz MVP
- Single-Feature MVP
- Landing Page MVP

## Prioritize

Use MoSCoW:

- Must Have
- Should Have
- Could Have
- Won't Have

Use RICE only if the prioritization is genuinely contested.

## Push

- Remove feature creep aggressively.
- Narrow target audience before adding features.
- Treat non-functional requirements as basic constraints, not an excuse
  to bloat scope.

## Output

MVP scope document with prioritized features, user stories, and success
metrics.
