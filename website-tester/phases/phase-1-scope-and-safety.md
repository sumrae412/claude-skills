# Phase 1: Scope and Safety

## Goal

Choose the test scope and establish safe execution boundaries.

## Decide

- full functional test vs quick smoke pass
- max pages
- screenshot / report outputs
- whether destructive operations are allowed

## Rules

- destructive ops are off by default
- staging is preferred when deletes are enabled
- public-only unless authentication support is added deliberately

## Output

Testing scope and safety settings for the run.
